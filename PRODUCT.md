# nom — Product Spec

> 一只活在你桌面上的宠物。它吃你 AI 客户端消耗的 token。

本文档定义 nom 是什么、为谁做、做到什么程度。技术细节请看 `CLAUDE.md`。

---

## 一句话定位

**nom 是一只悬浮在你桌面上的虚拟宠物，把你每天烧掉的 AI token 转化成一段陪伴感。**

## 愿景

每个开发者每天都在和 AI 客户端（Claude Code、Cursor、Codex CLI 等）对话，token 像水一样流走，但用户对"今天烧了多少、烧在哪"几乎没有具象感受。

nom 把这件抽象的事情变成一只**真的在你桌面上的小生物**：
- token 进来，它在屏幕一角"咀嚼"
- 攒到里程碑，它冒一句话出来
- 一天结束，它告诉你今天总共吃了多少

它不是仪表盘，是陪伴物。仪表盘让你皱眉，宠物让你笑一下。

## 目标用户

- 重度使用 AI 编码客户端的开发者（每天 >10K token）
- 喜欢桌面有点小装饰、不会觉得"挡事"的人
- macOS / Windows 用户

## 核心循环（MVP）

```
AI 客户端用 token  →  nom 监听到  →  桌面宠物动起来 + 状态变化  →  适时冒泡说话  →  用户被逗到 / 看到当日总量
```

四个动作，缺一不可。其它都是装饰。

## v1 MVP 范围（In Scope）

### 必须有
1. **桌面悬浮窗**：透明 / 无边框 / 置顶 / 不进任务栏，像 Shimeji 一样停在桌面某处。
2. **可拖动**：用户可以拖到屏幕任意位置；位置跨重启记忆。
3. **一只宠物**：原创角色，基础状态有 `idle` / `eating` / `sleeping` / `talking` 四种。第一版可以用 emoji/SVG 占位，最终用 PNG sprite。
4. **吃 token**：实时监听 Claude Code 的本地会话数据，每次有新 token 消耗触发"咀嚼"动画。
5. **冒泡对话**：
   - 点击宠物：手动触发一句话
   - 里程碑触发（每攒 10K token 说一句）
   - 闲置触发（>30min 无新事件 自言自语一句）
   - 启动 / 退出问候
6. **每日小结**：今天累计吃了多少 token、和昨天对比（用户点开看）
7. **退出方式**：右键宠物 → 菜单 → 退出
8. **跨平台**：macOS + Windows（Linux 不在 v1 承诺范围）
9. **本地优先 / 隐私**：所有 token 数据只在本机，绝不上传任何服务器

### 明确不做（Out of Scope，留给 v2+）
- ❌ **进化 / 长大**：v1 宠物形态固定，只切换状态动画。
- ❌ **多客户端**：v1 只接 Claude Code。Cursor / Codex CLI 留给 v2。
- ❌ **点击穿透**：v1 透明区域照样捕获鼠标（窗口尽量小，足迹小）；按像素穿透留 v2。
- ❌ **自动开机启动**：v1 用户每次手动启动；自动启动留 v2。
- ❌ **跨屏行走 / 自由走动**：v1 宠物在原地动，不会跑。
- ❌ **菜单栏 / 托盘图标**：v1 不做（用宠物的右键菜单做退出入口就够）。需要的话留 v2。
- ❌ **多宠物 / 换装 / 商店**：v1 一只宠物到底。
- ❌ **联机 / 排行榜 / 社交**：永远不做（隐私原则）。
- ⚠️ **AI 对话 / LLM 生成台词**：默认关闭，走本地预置台词。**可选**启用 (a) 本地 LLM（Ollama / mlx-lm，零网络）或 (b) 用户自己的 API Key（BYOK，nom 不内置任何官方 key）。任一模式都只发元数据（用量、时间），绝不发 prompt/response 内容。
- ❌ **自动更新**：v1 用户从 GitHub Releases 手动下新版。
- ❌ **代码签名**：v1 不签（用户首次打开会看到 macOS / SmartScreen 警告，README 说明绕过方式）。

## 分发

- **macOS**：`.dmg` 安装包
- **Windows**：`.exe` 安装器（NSIS）
- 都从 GitHub Releases 下载
- 不上 App Store / Microsoft Store（v1）

## 路线图（粗略）

- **v1.0** — 上面的 MVP，目标"能跑、有趣、能装"。
- **v1.1** — 自动开机启动、点击穿透、更多预置台词。
- **v1.2** — 更多状态动画（开心、生气、撑了）、菜单栏图标兜底。
- **v2.0** — 进化系统：吃够 N token 解锁新形态。
- **v2.1** — 多客户端适配（Cursor、Codex CLI、Aider 等）。
- **v2.2** — 跨屏行走、自由走动。

## 隐私原则（不可妥协）

1. nom 永不通过网络发送 token 数据。
2. nom 永不读取 token 内容（prompt / response 文本），只读用量数字。
3. 所有持久化数据存在用户本地（`~/.nom/`），用户随时可以 `rm -rf` 删掉。
4. README 第一行就写明这三条。
5. 应用不申请任何不必要的系统权限。

## 灵感

- **Shimeji** 桌面萌宠
- 早期 macOS 桌面宠物（如 BonziBuddy 的善意版）
- Tamagotchi 电子宠物机
- Codex CLI 的用量提示给了我们启发

## 不是

- 仪表盘 / 用量分析工具 — 这些事 nom 都能顺带做一点点，但不是主角
- AI 助手 / Copilot — nom 不"帮"你做事，它只是陪着你

## 成功标准（v1）

- 用户安装后能持续开着它至少一周
- 至少 30% 用户愿意分享截图给朋友看
- 没有任何隐私投诉
- macOS + Windows 各至少 100 次成功安装
