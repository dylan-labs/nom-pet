# nom — 下一步功能优先级（v1 后）

> 一次 idea-refine 会话的收敛结论。时间：2026-05-14。
> 完整发散过程见会话记录；本文只留可执行的结论。

## Problem Statement

**How might we** 让看到 nom 截图的开发者从"哈哈有点意思"变成"我要装一个"，并且装完之后舍不得关？

## Recommended Direction

把下一阶段重心放在 **"自传播 + 情感锚点"** 两件事上，**继续保持纯本地、零服务端**。

用户在 ideation 中已松口"可以联机"，但评估后认为联机是黑洞：服务端成本（账户、风控、合规、运维）对单人维护者过重，而 nom "零网络、零账户" 本身是真实的传播卖点（HN / Reddit / v2ex 受众吃这套）。**先用本地手段拿到 80% 联机能带来的传播效果，等装机量证明需求再考虑服务端**。

三个推荐方向，按投入产出比排序：

1. **战绩卡导出图 + 升级即分享 CTA** — 投入 1-2 天，把每个用户变成自来水
2. **命名 + 记忆碎片 + 悄悄话**（情感锚点三件套）— 让宠物从"装饰"变"我的"
3. **本地 LLM 真聊天**（差异化护城河）— nom 区别于所有静态桌面宠物的唯一硬卖点

## Key Assumptions to Validate

- [ ] **战绩卡真的会被分享** — 验证方式：先做最简版（黑底等宽字体 + 段位徽章 + 一句趣味标签），自己发 X / v2ex 看转发；若一周内无自然转发，说明设计不对，迭代视觉而不是放弃方向
- [ ] **"我的宠物有名字"会显著降低卸载率** — 验证方式：加上命名仪式后看周留存对比；样本不够就看自己用一周后会不会下意识叫它的名字
- [ ] **开发者真的在本地跑 LLM** — 验证方式：在 README 加一个非侵入式调研链接 / 在右键菜单藏"检测 Ollama"按钮，看点击率
- [ ] **段位升级是分享时刻** — 验证方式：升级 CTA 上线后看点击率 vs 主动截图率
- [ ] **不做联机不会被用户骂"功能少"** — 验证方式：看 GitHub Issue / Discord 反馈里"想要排行榜/好友"的比例

## MVP Scope（下一两个版本应该做什么）

### v0.0.21 — "传播版"
- [ ] 战绩卡导出图（Canvas 渲染，PNG 输出到 `~/Desktop/` 或剪贴板）
  - 日卡 / 周卡两种
  - 元素：宠物头像 + 段位徽章 + 今日/本周总 token + 一句趣味标签（"你今天最爱用 Python"之类，基于会话频次）
  - 视觉：黑底 + 等宽字 + 单色像素宠物，**程序员审美**，不要花哨
- [ ] 段位升级时金光特效后弹一个"截图分享"按钮（3 秒后自动消失）
- [ ] OBS 友好模式（右键菜单：宠物放大 1.5x，台词字号 +4pt）

### v0.0.22 — "情感版"
- [ ] 首次启动命名仪式（一个简单输入框；名字存 `~/.nom/state.json`）
- [ ] 命名后所有现有台词里 `${petName}` 插值
- [ ] 悄悄话功能（右键 → 写一句话 → 存到 `~/.nom/secrets.json`；按 1-7 天随机延迟随机冒出来）
- [ ] 记忆碎片（基于历史用量数据的预置模板台词："X 天前你一天吃了 Y token，那天你..."）

### v0.0.23+ — "差异化版"
- [ ] 检测 Ollama / LM Studio 在跑（端口探测）
- [ ] 一个非常窄的对话场景：右键"问问 nom" → 只回答关于今天用量的问题
- [ ] Prompt 设计：用宠物口吻 + 限制话题范围，避免 7B 模型暴露弱点

## Not Doing (and Why)

- **联机 / 排行榜 / 好友** — 服务端是新维度成本，先用战绩卡导出图拿 80% 传播效果。等装机量过万再评估
- **跨设备段位同步** — 同上，需要账户系统，留 v2+
- **皮肤工坊** — 需要 CDN + 审核，跳过
- **streak 连续天数** — 容易翻车成情感绑架（参考 Duolingo 的猫头鹰梗），先观察其他情感锚点效果再考虑
- **"3 天没开宠物会瘦"复活机制** — 同上，情绪绑架风险，等 streak 数据再说
- **自动开机启动** — 已在 PRODUCT.md v1.1 路线，按原计划
- **天气联动 / 节日彩蛋** — 锦上添花，不解决核心问题，进 backlog 而非下一版

## Open Questions

1. 战绩卡的"一句趣味标签"怎么生成？要不要做轻量分析（看 JSONL 里的 `cwd` 路径关键词推断在写什么项目）？还是只放硬数据避免猜错？
2. 命名仪式应该强制（首次启动不命名不能用）还是可跳过（用默认名 "nom"）？强制会拉高激活成本，可跳过会让一半人懒得起名
3. 本地 LLM 对话的 fallback：检测不到 Ollama 时，"问问 nom" 按钮显示还是隐藏？显示的话点击体验差，隐藏的话发现率为零
4. 战绩卡的尺寸：方形（适合 X / 朋友圈）还是 16:9（适合截屏背景）？做两套还是只选一种？
